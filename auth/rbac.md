# UK8S 授权管理

**注意：只有主账号才能进行授权管理操作。子账号或授信账号无法进行操作。**

## 开启授权管理

在新创建的集群下，授权管理会被默认开启。

如果是之前创建的集群，授权管理可能没有开启，需要到`授权管理`管理界面手动开启一下：

![]()

**如果没有开启授权管理，子账号将不会受到K8S RBAC权限控制，并且可以直接看到集群的管理凭证。**

## 角色

RBAC授权使用了K8S原生的权限管理，使用`ClusterRole`来对子账号进行授权。在此基础上，我们内置了一些默认角色：

| 角色       | 作用域         | 描述                                                         |
| ---------- | -------------- | ------------------------------------------------------------ |
| 管理员     | 集群、命名空间 | 可以操作集群或命令空间中的任何资源                           |
| 只读管理员 | 集群、命名空间 | 可以读取集群或命名空间中的任何资源                           |
| 开发者     | 命名空间       | 可以操作命名空间中的工作负载和配置等<br />无法查看、操作节点等运维资源<br />无法查看、操作secrets敏感资源 |
| 只读开发者 | 命名空间       | 可以查看命名空间中的工作负载和配置等<br />无法查看节点等运维资源<br />无法查看secrets敏感资源 |

如果上面的角色无法满足您的要求，您可以根据K8S的规范自定义角色，详细请参考[ClusterRole](https://kubernetes.io/zh-cn/docs/reference/kubernetes-api/authorization-resources/cluster-role-v1/)。

## 对一个子账号进行授权

在对一个子账号进行RBAC授权之前，即使我们给这个子账号赋予了RAM权限，其登录之后也无法访问UK8S中的资源，会收到下面的提示：

![]()

要想让子账号能够操作集群资源，需要对其进行RBAC授权。具体做法是把命名空间和上面的角色跟某个子账号进行绑定。

授权之后，子账号就能通过UK8S控制台或独立的凭证来访问UK8S中的资源了。子账号登录控制台可以看到自己的凭证：

![]()

这个凭证不同于集群的管理凭证，是为当前子账号单独颁发的，并且主账号随时可以对这个凭证进行撤销或刷新。

子账号在控制台的行为除了受到RAM授权的限制，也会受到RBAC授权的限制。例如，如果子账号只被授予了`default`这个命名空间的只读权限，那么他在控制台就无法查看别的命名空间的资源了：

并且，也无法通过控制台创建和修改任何资源：

![]()

子账号通过集群列表页进入`kubectl`控制台，也将会使用子账号自己的凭证：

![]()

> 注意：子账号在集群列表页进入`kubectl`控制台，会在您的集群下创建一个Pod用于访问集群，该Pod会在子账号退出kubectl页面后被销毁，但是会占用一些您的集群资源。
>
> 如果您不希望子账号使用`kubectl`功能，可以在[UK8S RAM 授权]()中禁用`GetUK8STerminalToken`这个API。

## 刷新凭证

每个凭证都有自己的有效时间。当凭证过期时，子账号将无法再通过凭证访问集群，这时候主账号需要对凭证进行刷新操作：

![]()

在凭证快要过期时，我们会提示您尽快刷新凭证。如果您不希望频繁刷新凭证，请选择尽量长的凭证过期时间。

## 修改角色

已经授权的用户是可以对它的命名空间或角色进行修改的：

![]()

修改之后新的角色会立即生效，并且凭证不会发生改变。

## 撤销授权

> 注意，如果注销了子账号，UK8S侧并不能自动撤销该账号的授权，此时凭证将会仍然可用，但是在授权列表中会看到账号已经被注销的提示。您需要手动对这个授权进行撤销。

您可以随时撤销某个子账号的授权：

![]()

在授权撤销之后，子账号将无法继续访问或操作UK8S中的资源了。他的凭证也将会无法使用。
